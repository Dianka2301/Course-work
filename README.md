# Food Recipe App

Food Recipe App — це сучасна кулінарна вебплатформа, що поєднує зручний робочий простір для щоденного готування, соціальну взаємодію між користувачами та інструменти штучного інтелекту (Google Gemini API).

Проєкт розроблено за клієнт-серверною архітектурою (React + Node.js) із використанням локальної бази даних SQLite.

---

# Технологічний стек

## Клієнтська частина
- React
- Vite
- React Router DOM

- Node.js
- Express

- bcrypt (хешування паролів)

# Основні можливості

## Профіль користувача
- Редагування імені, прізвища, email, біографії та завантаження аватара (із кропером зображень), що синхронізується з базою даних

---

## Інтерактивний каталог та обране
- Категорії з горизонтальним прокручуванням (адаптація для мобільних пристроїв і планшетів)
- Пошук за назвою або інгредієнтами
- Сортування за рейтингом, новизною та складністю
- Пагінація (15 рецептів на сторінку)
- Додавання рецептів в обране через кнопку ❤️

---

## Спільнота та обговорення
- Вкладені коментарі
- Оцінювання рецептів
- Фільтрація коментарів за кулінарною корисністю

---

- Генерація трьох рецептів за інгредієнтами
- Розрахунок КБЖВ (калорії, білки, жири, вуглеводи), часу приготування та складності
- Історія згенерованих рецептів

---

## Записник та модерація
- Приватні рецепти користувача
- Завантаження фотографій
- ШІ-модерація перед публікацією (перевірка якості, спаму та небезпечних інгредієнтів)

---

## Панель адміністратора
- Розділення заявок на окремі панелі
- ШІ-аналіз рецептів
- Можливість схвалення або відхилення рецептів

# Налаштування та запуск

## 1. Системні вимоги
- Node.js (версія 16 або вище)
- npm

---

## 2. Серверна частина

```bash
cd backend
npm install
```

Створіть `.env` файл у backend:

```env
PORT=4000
JWT_SECRET=ваша_секретна_строка_jwt
GEMINI_API_KEY=ваш_api_ключ_від_gemini
```

Запуск серверної частини:

```bash
node server.cjs
```

Сервер буде доступний за адресою:

```text
http://localhost:4000
```

---

## 3. Клієнтська частина

```bash
cd frontend
npm install
```

Створіть `.env` файл у frontend:

```env
VITE_API_URL=http://localhost:4000
```

Запуск клієнтської частини:

```bash
npm run dev
```

Застосунок буде доступний за адресою:

```text
http://localhost:5173
```

## Тестування

У проєкті реалізовано інтеграційне тестування серверної частини із використанням:

- Jest  
- Supertest  
- JWT для авторизованих запитів  
- окремої тестової SQLite бази даних  

### Тестове покриття

Тести покривають такі функціональні модулі:

- реєстрацію та авторизацію користувачів  
- перевірку JWT-автентифікації  
- отримання та оновлення профілю  
- створення приватних рецептів  
- роботу каталогу рецептів  
- систему обраних рецептів
- коментарі та вкладені відповіді  
- систему рейтингів  
- пагінацію рецептів  
- модерацію рецептів адміністратором  
- генерацію сповіщень  
- перевірку схожих рецептів за інгредієнтами  

### Тестова база даних

Для тестування використовується ізольована база даних `test-database.db`, яка автоматично створюється перед запуском тестів та видаляється після їх завершення.
